# Security Policy

## Supported versions

Security fixes are currently targeted at the latest `0.1.x` state of the repository.

## Reporting a vulnerability

Please do not open a public issue for a suspected security problem.

Use one of these routes instead:

- GitHub Security Advisories or private vulnerability reporting, if enabled for the repository
- Direct private contact with the maintainers through GitHub

When reporting, include:

- A short description of the issue and affected area
- Reproduction steps or a proof of concept
- Expected impact
- Any suggested mitigation, if you have one

Please avoid attaching personal local archives, secrets, or unnecessary sensitive data. If private reporting is not available, open a minimal public issue that requests a private contact path without disclosing exploit details.
